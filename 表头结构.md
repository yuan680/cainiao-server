## 脚本关键列

| 变量 | 对应列名 | 类型 | 说明 |
|------|----------|------|------|
| `COL_MAIL_NO` | 物流单号 | 文本 | 查询依据 |

| `COL_TIME` | 更新时间 | 日期 | 记录查询时间戳 |

## 查询分流逻辑

```
读取全部 LP 开头行
  ↓
判断每行物流状态值：
  ├─ 空白 / "待查询" / "查询中..." / "无数据" → emptyRows
  └─ 已有其他值 → hasStatusRows
  ↓
选择查询范围：
  ├─ emptyRows > 0 → 只查 emptyRows（补空白行）
  ├─ emptyRows = 0 → 全量刷新 hasStatusRows（从头查全部）
  └─ 循环直到 allPending 最后一条
```

| 场景 | 行为 | 说明 |
|------|------|------|
| 全部空白 | 查全部 | 从头到尾刷一遍 |
| 部分有值部分空白 | 只查空白行 | 补缺，不碰已有状态的行 |
| 全部有值 | 查全部 | 全量刷新，覆盖旧状态 |
