# WPS 多维表集成物流轨迹查询指南

**核心发现：菜鸟全球物流公开 API 无需鉴权 ✅**

已确认 `https://global.cainiao.com/global/detail.json?mailNos=xxx&lang=zh-CN` 可直接调用，**不需要** AppKey、AppSecret、注册开发者。

---

## 推荐方案：本地桥接服务 + 按钮「发送 HTTP 请求」

WPS 多维表的「按钮」字段支持「发送 HTTP 请求」动作，可调用本地 Python 服务查询物流，并将结果写回表格。

### 工作流程

```
WPS 多维表「更新」按钮 → HTTP GET 请求 →
本地服务 http://localhost/query?mailNo=... →
调用菜鸟公开 API → 返回 JSON → WPS 自动填入「物流状态」列
```

### 第一步：一键安装（只需执行一次）

**右键** `install.bat` → **以管理员身份运行**，脚本会自动完成：

1. ✅ 设置端口转发 `80 → 58080`（解决 WPS 禁止端口号问题）
2. ✅ 添加开机自启（以后开机自动运行，无需任何操作）
3. ✅ 立即启动后台服务

完成后直接进入第二步，**无需手动启动服务**，也**无需每次重启电脑后重新设置**。

> **卸载**：右键 `uninstall.bat` → 以管理员身份运行即可完全移除。

### 验证服务是否运行

在浏览器打开以下地址，应返回 JSON：
```
http://localhost/query?mailNo=LP00812637173551
```

> 如果浏览器打不开，可能是 80 端口被占用。此时可改用 `start_server.bat` 手动启动服务。

### 第二步：WPS 多维表配置（3 列）

在 WPS 多维表中创建以下字段：

| 字段名 | 字段类型 | 说明 |
|--------|----------|------|
| `物流单号` | **文本** | 输入待查询的运单号（如 LP00812637173551） |
| `更新` | **按钮** | 配置为「发送 HTTP 请求」动作 |
| `物流状态` | **文本** | 自动接收查询结果 |

### 第三步：配置按钮的 HTTP 请求

点击「更新」按钮字段进入配置：

1. **触发动作**：选择 **「发送 HTTP 请求」**
2. **请求方式**：`GET`

3. **请求 URL**：填入以下任意一种方式即可

   **方式 A（推荐）：直接写在 URL 里**
   ```
   http://localhost/query?mailNo={物流单号}
   ```
   > `{物流单号}` 是跨列引用占位符，WPS 会自动替换为该行的运单号

   **方式 B：分拆填写「查询参数」**
   - 请求 URL：`http://localhost/query`
   - 查询参数（键值对）：
     | 参数名 | 参数值 |
     |--------|--------|
     | `mailNo` | `{物流单号}` |

4. **请求头（Headers）**：此接口无需特殊请求头，保持默认空即可。如需填写，可添加：
   - `Content-Type` → `application/json`（本 GET 请求实际不需要，留空亦可）
   - **无需** `Authorization` / `AppKey` 等鉴权字段

5. **返回格式**：`JSON`

6. **数据写入**（将返回字段映射到表格列）：
   - 目标列：`物流状态`
   - 映射值：`$data[0].status + " | " + $data[0].latestEvent`
   
   如果 WPS 多维表不支持表达式拼接，可选择只显示状态：
   - 目标列：`物流状态`
   - 映射值：`$data[0].status`

### 第四步：测试

在「物流单号」列输入 `LP00812637173551`，点击「更新」按钮 → 几秒后「物流状态」列自动填入结果。

## API 返回值参考

`GET http://localhost/query?mailNo=LP00812637173551` 返回：

```json
{
  "code": 0,
  "data": [
    {
      "mailNo": "LP00812637173551",
      "status": "已签收",
      "statusCode": "signed",
      "origin": "中国",
      "dest": "美国",
      "latestTime": "2024-01-15 14:30:00",
      "latestEvent": "快件已签收",
      "eventCount": 12
    }
  ]
}
```

WPS 多维表可引用的字段：

| 路径 | 说明 |
|------|------|
| `$data[0].status` | 物流状态文本 |
| `$data[0].latestEvent` | 最新事件描述 |
| `$data[0].latestTime` | 最新事件时间 |
| `$data[0].origin` | 发件地 |
| `$data[0].dest` | 目的地 |
| `$data[0].eventCount` | 轨迹事件总数 |

## 备用方案 C：WPS JS 宏调用本地 EXE

适用于 WPS 桌面版（支持 ActiveX）。WPS 多维表**网页版不支持**。

```javascript
// WPS JS 宏 - 查询物流轨迹
function QueryTrack() {
    var sheet = ActiveSheet;
    var row = ActiveCell.Row;
    var mailNo = Cells(row, 1).Value;  // 假设物流单号在 A 列
    var exePath = "F:\\cainiao_track\\cainiao_track.exe";

    var shell = new ActiveXObject("WScript.Shell");
    var cmd = exePath + " " + mailNo + " --json";
    var exec = shell.Exec(cmd);
    var stdout = exec.StdOut.ReadAll();

    if (exec.ExitCode === 0) {
        var result = JSON.parse(stdout);
        Cells(row, 3).Value = result.data[0].status;  // 写入 C 列
    } else {
        Cells(row, 3).Value = "查询失败";
    }
}
```

| 写入多列 | ❌ 只能写一列 | ✅ 状态、事件、时间、目的地均可分列 |
| 跳过已完结 | ❌ 每次全部查询 | ✅ 智能跳过已签收订单 |
| 自定义错误处理 | ❌ | ✅ 失败行标注具体原因 |
| 输出格式控制 | ❌ 固定 JSON 映射 | ✅ 自由拼接文本 |

### 第二步：创建脚本

1. 打开你的 WPS 多维表格
2. 点击顶部菜单 **「脚本」→「脚本编辑器」**（如找不到，点「更多」展开）
3. 在脚本编辑器中，点击 **「新建脚本」** (➕)
4. 将 `wps_macro_tracking.js` 的全部内容粘贴进去
5. 按 **Ctrl+S** 保存，脚本名称填 `菜鸟物流查询`
6. 关闭脚本编辑器

📁 脚本源文件位置：`F:\cainiao_track\wps_macro_tracking.js`

### 第三步：配置按钮（每行一个，逐行查询）

这是最便捷的日常使用方式。参照你截图中已有的按钮字段来配置：

#### 3.1 确保已有字段（列）

你的表格目前有 `快递单号`、`物流公司`、`按钮`。建议再添加以下文本字段用于接收查询结果：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `快递单号` | **文本** | ✅ 已有，读取单号来源 |
| `物流状态` | **文本** | 建议添加，自动填入状态 |
| `物流公司` | **文本** | ✅ 已有（可选） |
| `更新时间` | **文本** | 建议添加，查看最后查询时间 |
| `按钮` | **按钮** | ✅ 已有 |

> 如果不想加新字段也没关系，脚本只会更新脚本中已配置且表格中存在的字段。

#### 3.2 按钮配置步骤

参照你的截图（图2），按以下设置：

1. **点击按钮字段的列头** → 打开字段配置面板
2. 在「点击按钮时」区域：
   - **动作选择**：`执行 AirScript 脚本` ✅（你截图已选）
   - **选择脚本文件**：点「选择」→ 选 `菜鸟物流查询`
   - **选择函数**：选 `查询当前行(record)`（⭐ 这是专门为按钮新增的函数，只处理当前行）
3. 关闭配置面板 ✅

#### 3.3 使用效果

配置完成后，每行都会出现一个按钮。点击某行的按钮：
- 该行的 `快递单号` 被读取 → 调用本地 API
- 结果写回该行的各列（物流状态、更新时间等）
- 其他行不受影响

**注意**：如果点击后弹出"运行流程失败"的红色提示（如图1），最常见的原因是：
- 脚本文件名或函数名填写有误 → 重新选一次「菜鸟物流查询.查询当前行」
- API 服务未启动 → 先双击 `start_server.bat` 启动服务器

### 脚本配置修改

**⭐ 重点：你的列名是「快递单号」，不是「物流单号」**。在脚本编辑器里找到脚本顶部的配置区域，把第一行改为：

```javascript
var FIELD_MAIL_NO   = "快递单号";   // ← 改成你表格里实际的列名
```

其他列名如果与表格不一致也同理修改。配置区域完整如下：

```javascript
// API 地址（默认端口 58080）
var API_BASE = "http://localhost:58080";

// 表头字段名（务必与你表格中的字段名一致）
var FIELD_MAIL_NO   = "快递单号";   // ⬅️ 你的列名叫"快递单号"
var FIELD_STATUS    = "物流状态";   // ⬅️ 如果你有这列就保留，没有就保持原样
var FIELD_EVENT     = "最新事件";   // 如不想要这列，脚本会自动跳过
var FIELD_TIME      = "最新时间";
var FIELD_DEST      = "目的地";
var FIELD_UPDATED   = "更新时间";

// 已完结状态列表（匹配的行不再重复请求）
var FINAL_STATUSES = ["已签收", "已签收（本人）", "已签收（他人代签）",
                      "异常签收", "派送失败", "退回", "销毁"];

// 是否跳过已完结订单
var SKIP_FINAL = true;
```

> 如果已用 `install.bat` 设置了端口转发（80 → 58080），可将 `API_BASE` 改为 `http://localhost`（不加端口号），这样 WPS 兼容性更好。

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 按钮点击无反应 | 本地服务未启动 | 先运行 `python cainiao_server.py` |
| 返回空数据 | 运单号不存在 | 到菜鸟官网验证 |
| 网络错误 | 无法访问 global.cainiao.com | 检查网络/代理 |
| 端口被占用 | 58080 已被使用 | 关闭占用程序，或修改 `--port` |
| $data[0] 无法引用 | WPS 映射语法不同 | 尝试 `$data` 或仅 `data[0].status` |
